## КЛЮЧОВІ БІЗНЕС-ПИТАННЯ

### Phase 1: Per-Market DiD Analysis
**Що відбувається з попитом, коли препарат відсутній в аптеці?**

```
Покупець приходить за НУРОФЕН таблетками, а їх немає:
├─ Купує ІБУПРОФЕН таблетки в цій же аптеці? (INTERNAL)
└─ Купує НУРОФЕН в іншій аптеці? (LOST)
```

Цей аналіз виконується для кожного локального ринку окремо і дає базові метрики субституції per market.

---

### Phase 2: Cross-Market Substitution Coefficients (основне дослідження)

**Ключові дослідницькі питання:**

1. **Загальна тенденція субститованості:**
   > Чи мають досліджувані препарати загальну тенденцію субститованості
   > не на 1 локальному ринку, а на ВСІХ досліджуваних локальних ринках?

2. **Можливість визначення коефіцієнта:**
   > Чи можна визначити коефіцієнт субститованості препарату,
   > виходячи з даних його субститованості з різних локальних ринків?

3. **Методологія розрахунку:**
   > Як правильно розрахувати такий коефіцієнт субститованості?

4. **Статистичні вимоги:**
   > Яким статистичним вимогам повинен відповідати препарат
   > для надійного визначення коефіцієнта субститованості?

**Очікуваний результат:**

```
Препарат X на 50+ ринках:
├─ SHARE_INTERNAL стабільно високий → SUBSTITUTABLE (можна оптимізувати SKU)
├─ SHARE_INTERNAL стабільно низький → CRITICAL (обов'язково тримати)
└─ SHARE_INTERNAL нестабільний → MODERATE (аналізувати індивідуально)
```

## МЕТРИКИ ТА ІНТЕРПРЕТАЦІЯ

### Per-Market метрики

| Метрика | Формула | Інтерпретація |
|---------|---------|---------------|
| **SHARE_INTERNAL** | INTERNAL_LIFT / TOTAL_EFFECT | Частка попиту, що залишилась в аптеці |
| **SHARE_LOST** | LOST_SALES / TOTAL_EFFECT | Частка попиту, що пішла до конкурентів |
| **SHARE_SAME_NFC1** | LIFT_SAME_NFC1 / INTERNAL_LIFT | Частка, що обрала ту ж форму випуску |

### Cross-Market метрики

| Метрика | Формула | Інтерпретація |
|---------|---------|---------------|
| **MARKET_COVERAGE** | N_markets_with_drug / N_total_markets | % ринків, де присутній препарат |
| **MEAN_SHARE_INTERNAL** | mean(SHARE_INTERNAL across markets) | Середнє значення субституції |
| **STD_SHARE_INTERNAL** | std(SHARE_INTERNAL across markets) | Варіативність субституції |
| **CI_95_LOWER/UPPER** | mean ± 1.96 * (std / sqrt(N)) | 95% довірчий інтервал |

## КЛАСТЕРИЗАЦІЯ ЗА ПОКРИТТЯМ

| Кластер | Критерій | Бізнес-значення |
|---------|----------|-----------------|
| **COVERAGE_100** | Присутній на 100% ринків | Універсальний препарат |
| **COVERAGE_90** | Присутній на 90-99% ринків | Широко поширений |
| **COVERAGE_80** | Присутній на 80-89% ринків | Поширений |
| **COVERAGE_50** | Присутній на 50-79% ринків | Регіональний |
| **COVERAGE_LOW** | Присутній на <50% ринків | Нішевий |

**Статистична значущість:** Препарати з низьким coverage мають менше точок даних → ширші довірчі інтервали → менш надійні висновки.

## КЛАСИФІКАЦІЯ ПРЕПАРАТІВ

### Трирівнева класифікація

| Категорія | Критерій | Бізнес-рішення |
|-----------|----------|----------------|
| **CRITICAL** | CI_95_UPPER(SHARE_INTERNAL) < threshold_low | Обов'язково тримати в наявності |
| **SUBSTITUTABLE** | CI_95_LOWER(SHARE_INTERNAL) > threshold_high | Можна оптимізувати SKU |
| **MODERATE** | Інші випадки | Аналізувати індивідуально |

**Пороги визначені в:** `project_core/did_config/`

### Логіка класифікації

```
IF upper bound довірчого інтервалу < low_threshold:
    → Навіть в найкращому випадку субституція слабка
    → CRITICAL

ELIF lower bound довірчого інтервалу > high_threshold:
    → Навіть в найгіршому випадку субституція сильна
    → SUBSTITUTABLE

ELSE:
    → Невизначеність занадто велика
    → MODERATE
```

## БІЗНЕС-РЕКОМЕНДАЦІЇ

### За результатами аналізу:

1. **CRITICAL препарати** — не допускати stock-out across all markets:
   - Централізоване управління запасами
   - Пріоритет при розподілі

2. **SUBSTITUTABLE препарати** — можна оптимізувати SKU:
   - Зменшити кількість позицій в групі
   - Централізувати на 2-3 найпопулярніші варіанти

3. **MODERATE препарати** — потребують індивідуального аналізу:
   - Переглянути по ринках з найбільшими втратами
   - Можливо, різна стратегія для різних регіонів

## ГЛОСАРІЙ

### Терміни мульти-ринкового аналізу

| Термін | Визначення |
|--------|------------|
| **Local Market** | Географічна зона навколо цільової аптеки (= дані одного CSV файлу) |
| **Target Pharmacy** | Аптека, для якої проводиться аналіз (CLIENT_ID) |
| **Coverage** | Частка ринків, де препарат присутній в асортименті |
| **Cross-Market Stability** | Узгодженість метрики SHARE_INTERNAL across markets |
| **CI (Confidence Interval)** | Довірчий інтервал для оцінки невизначеності |

### Терміни DiD аналізу

| Термін | Визначення |
|--------|------------|
| **DiD** | Difference-in-Differences — метод оцінки ефекту |
| **Stock-out** | Період відсутності препарату (Q=0 ≥ 1 тиждень) |
| **PRE-період** | Тижні до stock-out (baseline) |
| **LIFT** | Додаткові продажі: max(0, ACTUAL - EXPECTED) |
| **MARKET_GROWTH** | Коефіцієнт росту ринку |

### Терміни даних

| Термін | Визначення |
|--------|------------|
| **CLIENT_ID** | ID цільової аптеки (з назви файлу Rd2_{ID}.csv) |
| **ORG_ID** | ID аптеки, що здійснила продаж |
| **INN_ID** | Код групи діючої речовини |
| **NFC1** | Широка категорія форми випуску |

### NFC1 категорії

```
ORAL_GROUP (взаємозамінні):
├─ Пероральные твердые обычные (таблетки, капсули)
├─ Пероральные жидкие обычные (сиропи)
└─ Пероральные твердые длительно действующие

EXACT_MATCH (тільки на себе):
├─ Парентеральные обычные (ін'єкції)
├─ Местно действующие (мазі, гелі)
├─ Ректальные системные (свічки)
└─ Інші форми
```
